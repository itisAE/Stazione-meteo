Esempio di utilizzo:

Ecco un esempio completo di come inviare i dati meteorologici:

curl -X POST https://api.stazione-meteo.itis-delpozzo.it/weather-data \ -H "Authorization: Bearer {{JWT_TOKEN}}" \ -H "Content-Type: application/json" \ -d '{ "station_id": "{{STATION_ID}}", "timestamp": "2025-09-13T14:30:00Z", "location": { "latitude": 44.5675, "longitude": 7.4823, "altitude": 425 }, "data": { "temperature": { "value": 23.5, "accuracy": 95.0, "unit": "celsius" }, "humidity": { "value": 68.2, "accuracy": 90.0, "unit": "percent" }, "pressure": { "value": 1013.25, "accuracy": 98.5, "unit": "hPa" } }, "sensor_info": { "model": "Davis Vantage Pro2", "firmware_version": "1.2.3" } }'

Parametri fondamentali:

  • station_id: Identificativo univoco della tua stazione
  • accuracy: Percentuale di accuratezza per ogni sensore (usa il nostro calcolatore!)
  • timestamp: Data e ora in formato ISO 8601 UTC
  • location: Coordinate precise della tua stazione

Calcolatore di Accuratezza:

Per determinare la percentuale di accuratezza corretta per i tuoi sensori, utilizza il nostro calcolatore online. Considera:

  • Modello del sensore
  • Precisione dichiarata dal produttore
  • Età dell'installazione
  • Data ultima calibrazione
  • Condizioni ambientali attuali
Limite di utilizzo: Il tuo token permette fino a 100 richieste per ora. Se hai bisogno di un limite maggiore, contattaci.

La tua contribuzione conta:

Ogni dato che invii attraverso le API contribuisce a:

  • Nowcasting migliorato: Previsioni più accurate a 2-3 ore
  • Risoluzione locale: Dati su scala micro-climatica
  • Validazione ML: Perfezionamento degli algoritmi predittivi
  • Rete distribuita: Copertura meteorologica capillare del territorio
